refactor: Download CSV from URL instead of committing to repo

**Changes:**

1. **CSV Download from URL**
   - Downloads bharatpin CSV from GitHub (jeet308/bharatpin)
   - Default URL: https://raw.githubusercontent.com/jeet308/bharatpin/main/src/bharatpin/data/pincodes.csv
   - Saves to /tmp/bharatpin_pincodes_2026.csv
   - Reuses existing file if present

2. **Removed Large Files from Repository**
   - Deleted bharatpin_pincodes_2026.csv (21.5 MB)
   - Deleted Datagov_Pincode_Boundaries.geojson (87 MB)
   - Moved to parent directory (not tracked by git)

3. **Updated .gitignore**
   - Added *.csv
   - Added *.geojson
   - Prevents accidental commits of large data files

4. **Updated CSV Ingestion Service**
   - Added downloadCSV() method (similar to GeoJSON download)
   - Checks for existing file before downloading
   - Uses HTTPS with proper error handling
   - Configurable via CSV_DATA_URL environment variable

5. **Updated Initialization Service**
   - Removed csvPath parameter
   - Service now handles download automatically
   - Cleaner API

6. **Updated Documentation**
   - CSV_INGESTION_GUIDE.md reflects URL-based approach
   - IMPLEMENTATION_SUMMARY.md updated
   - Environment variables documented

**Benefits:**
- Repository stays small (no 100+ MB data files)
- Data always fresh (downloads latest from source)
- Consistent with GeoJSON download approach
- Environment-specific URLs supported
- Production can use different data source if needed

**Environment Variables:**
```
CSV_DATA_URL - URL to download CSV from (optional)
```
