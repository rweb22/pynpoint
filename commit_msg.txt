feat: add POST /pincodes/locate endpoint for point-in-polygon pincode lookup

- Added new endpoint POST /api/v1/pincodes/locate for exact coordinate-to-pincode matching
- Uses PostGIS ST_Contains for point-in-polygon boundary check (vs ST_Distance for nearest)
- Distinct from /reverse-geocode which finds nearest pincodes by distance
- Returns pincode boundary that contains coordinates, or null if outside all boundaries
- Added LocatePincodeDto and LocatePincodeResponseDto
- Fixed RedisCacheService access in PincodeCacheService using getClient() method
- Updated API discovery page to reflect both endpoints

Use cases:
- /locate: Find pincode from GPS coordinates (exact boundary match)
- /reverse-geocode: Find nearest pincodes by distance (distance-based search)
